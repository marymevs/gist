## Review Passes

### Pass 1: Information Architecture -- 9/10

Clear reading order: masthead -> weather -> rhythms -> lede -> columns. Page 1 is information (your day). Page 2 is meaning (your life). Left columns hold structured data. Right columns hold relational/emotional content. The lede is the front-page story of your day -- not a summary, an *editorial*. The Fax Back section turns a one-way output into a two-way communication device. The personal countdown in the rhythms bar had no data model field (resolved: adding to user prefs).

### Pass 2: Interaction State Coverage -- 7/10

The samples show "perfect days" with rich calendars and multiple emails. Undefined states that need prompt guidance:

| Content Area | Sparse Data | No Data |
|---|---|---|
| Schedule | 1-2 events: does the column feel empty? | Zero events: what fills the space? |
| Notifications | 1-2 notifications: adequate? | No email/calendar connected |
| News | News API down: fallback content? | User opted out of news |
| People | Only 1 person in data | No relationship signals |
| Rhythms | No countdown configured | Weather API down |
| Page 2 editorial | User has no major project | Minimal user context |
| Fax Back | N/A (static structure, always present) | N/A |

The LLM prompt needs explicit guidance on minimum viable content and graceful filling when data is sparse.

### Pass 3: User Journey and Emotional Arc -- 10/10

The emotional design is masterful:

| Step | User Does | Feels | Design Supports |
|---|---|---|---|
| 1 | Receives fax/opens email | Anticipation, ritual | Masthead ceremony, "Vol. I No. 47" |
| 2 | Reads weather + rhythms | Grounded in time | Moon/season/daylight = something larger |
| 3 | Reads the lede | Seen, understood | "Good Morning, Riley" -- personal editorial |
| 4 | Scans schedule | Prepared, in control | Coaching notes per event |
| 5 | Reads notifications | Informed without overwhelm | Curated with *why it matters to you* |
| 6 | Reads People | Connected, accountable | "Dad texted twice... you replied once" |
| 7 | Turns to page 2 | Shift to reflection | Layout change signals new mode |
| 8 | Reads Body + Mind | Cared for, validated | Acknowledges struggle without platitude |
| 9 | Fills Morning Intention | Intentional | Physical writing = embodied commitment |
| 10 | Marks up Fax Back | Closure, agency | Checkboxes + write lines |
| 11 | Faxes it back | Loop closed | "Your agents process overnight" |

Closing quotes are personalized: "The Gist, for Riley." The product speaks to you.

### Pass 4: AI Slop Risk -- 9/10

Zero hard rejections triggered. This is anti-slop -- a newspaper, not a SaaS dashboard. Zero shadows, zero cards, zero gradients. Pure typography and rules. The highlight box left border (#8 on the slop blacklist) earns its existence as a genuine editorial pull-quote pattern. Classification: APP UI (print artifact). All App UI rules pass.

| Target | CSS Support | Status |
|---|---|---|
| Print/Fax (US Letter) | Full | Ready (samples designed for this) |
| PDF (browser print) | Full | Ready (@page rules present) |
| Web /today view | Full | Needs mobile breakpoint below 768px |
| Email (Gmail) | No CSS vars/flex/grid/@import | Needs table-based fallback |
| Email (Apple Mail) | Partial | Needs testing |

Gaps: (1) Gmail strips CSS variables, flexbox, grid, and @import -- needs separate table-based email template. (2) Two-column layout at 7in breaks on phones -- needs stacked single-column below 768px. (3) pt units don't respect browser font-size prefs -- web version should use rem. (4) #a8a29e on #fffdf9 is 2.8:1 contrast -- fails WCAG AA for small text on web (acceptable for print labels).

## What's Great (Don't Touch)

- **Two-page structure.** Page 1 is the briefing (information). Page 2 is the reflection (intention). This is the product's soul.
- **The lede.** A personalized editorial headline + paragraph that sets the emotional tone. Not a summary -- an *editorial*.
- **The rhythms bar.** Moon, season, daylight, personal countdown. Grounds you in time larger than today.
- **Typography hierarchy.** 46pt masthead -> 16pt lede -> 8.5pt body -> 5.5pt labels. Perfectly scaled for print density.
- **Closing quotes.** "The Gist, for Riley" -- the product speaks to you personally. Not generic inspirational quotes.
- **Fax Back.** Turns one-way output into two-way communication. Checkboxes + write lines = physical commitment.

## What Needs Building

- **New output design system in DESIGN.md.** Document Fraunces/IBM Plex/warm amber as "Output Template" alongside existing web app system.
- **New template function.** Replace faxTemplate.ts and emailTemplate.ts with newspaper-layout templates matching these samples.
- **Extended Zod schema.** New sections: lede (kicker + headline + paragraph), rhythms (moon, season, light, countdown), people (name + nudge pairs), bodyMind (editorial prose), moonHighlight, closingQuote (quote + attribution), goodNews.
- **Email-safe fallback.** Table-based version of the newspaper template for Gmail/Outlook while preserving editorial feel.
- **User model additions.** countdown: { label: string, targetDate: string } in prefs, gistIssueCount: number on user doc.
- **Prompt overhaul.** Rewrite Claude prompt to produce the editorial voice shown in samples. Current prompt produces functional summaries. Samples produce warmth, accountability, humor.

## Key Risks

- **Email rendering.** Gmail strips CSS variables, flexbox, grid, and @import. The newspaper layout needs a completely separate email-safe template. This is the hardest implementation challenge.
- **Sparse data days.** A user with 1 calendar event and no email connected gets a thin Gist. The prompt needs explicit minimum-content guidance.
- **LLM voice consistency.** The editorial voice in these samples is extraordinary. Getting Claude to consistently produce this quality requires careful prompt engineering with few-shot examples.

## Not In Scope

- Dark mode output (print-first; dark mode is a web concern)
- Evening Gist template (deferred per TODOS.md, P3)
- Fax Back OCR processing (inbound pipeline is separate work)
